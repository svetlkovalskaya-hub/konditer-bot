# Помощник кондитера — Telegram-бот для записи клиентов

Бот берёт на себя рутину записи клиентов на торты, пряники, зефир и другие десерты. Создан для домашних кондитеров и небольших мастерских, у которых пока нет готовой CRM.

**🤖 Попробовать бота:** [@Mykonditerbot](https://t.me/Mykonditerbot)

## Зачем этот бот

Домашний кондитер обычно ведёт заказы в блокноте, мессенджерах и голове. Это работает, пока заказов мало. Когда их становится больше, начинается хаос: забытые даты, двойные брони, переписка в нескольких чатах, дизайн в одном месте, адрес доставки — в другом.

Помощник кондитера собирает всё в одном окне Telegram:
- клиент выбирает изделие и вид торта с фото,
- выбирает удобную дату и время,
- указывает адрес доставки или самовывоз,
- добавляет комментарий и фото-примеры,
- получает подтверждение заказа.

Кондитер видит список заказов, получает уведомления и управляет записями через кнопки в админ-панели.

## Что умеет бот

### Для клиента

- `/start` — приветствие и кнопка «📝 Записаться».
- Выбор изделия из каталога: торт, пряники, зефир, макарон, капкейки.
- Для тортов — выбор вида из списка с фото.
- Для Наполеона и Медовика — выбор вкуса: классический или шоколадный.
- Inline-календарь для выбора даты.
- Готовые слоты времени или свободный ввод.
- Адрес доставки или самовывоз.
- Комментарий: дизайн, вкус, аллергены, пожелания.
- Прикрепление фото-примеров (до 5 штук).
- Предпросмотр заказа перед подтверждением.
- `/myorders` — список своих заказов.
- Перенос записи на другую дату.

### Для кондитера

- Уведомление о новом заказе с пометкой источника: Telegram, MAX или ручная запись.
- Кнопки в админ-панели:
  - 📋 Список заказов
  - 🔍 Найти по имени
  - 📞 Поиск по телефону
  - ➕ Записать заказ от имени клиента
  - ❌ Удалить заказ
  - 🔄 Перенести запись
- Утреннее приветствие в 9:00 со списком заказов на сегодня и завтра.
- Защита от перебронирования: на одну дату можно принять ограниченное число заказов.
- Блокировка дат, чтобы клиенты не записывались на занятые дни.

## Стек

- Node.js (JavaScript, CommonJS)
- `express` — мини-сервер для healthcheck и сброса webhook
- `node-telegram-bot-api` — Telegram-бот
- `better-sqlite3` — локальная база данных
- `node-cron` — утренние напоминания
- `dotenv` — переменные окружения

## Локальный запуск

### 1. Создай бота и получи токен

1. Напиши [@BotFather](https://t.me/botfather).
2. Отправь `/newbot` и следуй инструкциям.
3. Скопируй токен.

### 2. Узнай свой Telegram ID

Напиши [@userinfobot](https://t.me/userinfobot) или [@getidsbot](https://t.me/getidsbot). Этот ID нужен для роли администратора.

### 3. Настрой окружение

Скопируй `.env.example` в `.env`:

```bash
cp .env.example .env
```

Открой `.env` и заполни:

```env
TELEGRAM_BOT_TOKEN=токен_из_@BotFather
ADMIN_TELEGRAM_ID=твой_telegram_id
PORT=3000
NODE_ENV=development
```

### 4. Установи зависимости

```bash
npm install
```

### 5. Добавь фото тортов

1. Создай папку `uploads/cake-photos/`.
2. Положи туда фото видов тортов.
3. Открой `src/config/cakeTypes.js` и проверь пути.

Пример:

```js
{ name: 'Наполеон', photoFile: 'uploads/cake-photos/наполеон.jpg', flavors: ['Классический', 'Шоколадный'] }
```

### 6. Запусти бота

```bash
npm start
```

В консоли должно появиться:

```
Telegram-бот запущен
Сервер запущен на порту 3000
```

### 7. Проверь работу

1. Найди бота в Telegram и отправь `/start`.
2. Пройди сценарий заказа до конца.
3. Убедись, что админу пришло уведомление.
4. Проверь админ-панель через кнопки в меню.

## Деплой на сервер (VPS / Beget / аналоги)

Рекомендуется Ubuntu 22.04 или 24.04 LTS.

### 1. Подключись к серверу

```bash
ssh root@IP_АДРЕС_СЕРВЕРА
```

### 2. Установи Node.js 18+ или 22 LTS

```bash
# Node.js 22 LTS через NodeSource
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt-get install -y nodejs
```

### 3. Установи git и PM2

```bash
apt update && apt install -y git
npm install -g pm2
```

### 4. Склонируй проект

```bash
cd /opt
git clone https://github.com/svetlkovalskaya-hub/konditer-bot.git
cd konditer-bot
npm install
```

### 5. Создай `.env`

```bash
nano .env
```

Пример:

```env
NODE_ENV=production
PORT=3000
TELEGRAM_BOT_TOKEN=токен_из_@BotFather
ADMIN_TELEGRAM_ID=твой_telegram_id
```

### 6. Скопируй фото тортов

Папка `uploads/` не входит в репозиторий. Загрузи фото на сервер отдельно:

```bash
mkdir -p /opt/konditer-bot/uploads/cake-photos
```

Затем скопируй файлы любым удобным способом: `scp`, FTP или через панель хостинга.

### 7. Запусти бота через PM2

```bash
pm2 start src/index.js --name konditer-bot
pm2 save
pm2 startup
```

Выполни команду, которую выведет `pm2 startup`.

### Управление ботом на сервере

```bash
cd /opt/konditer-bot

pm2 status                # статус бота
pm2 logs konditer-bot     # логи в реальном времени
pm2 logs --lines 50       # последние 50 строк логов
pm2 restart konditer-bot  # перезапуск
pm2 stop konditer-bot     # остановка
```

### Обновление кода

```bash
cd /opt/konditer-bot
git pull
npm install
pm2 restart konditer-bot
```

## Важные нюансы

### Фото тортов

Папка `uploads/cake-photos/` не сохраняется в git. При каждом деплое на новый сервер копируй её вручную.

### Часовой пояс

Утреннее приветствие запускается в 9:00 по системному времени сервера. Если сервер не в московском часовом поясе — скорректируй время или настрой `TZ=Europe/Moscow` перед запуском.

### Webhook

Бот работает через polling. Если раньше был настроен webhook — сбрось его через эндпоинт:

```
http://IP_СЕРВЕРА:PORT/reset-webhook
```

Или вручную через Telegram API:

```
https://api.telegram.org/bot<TOKEN>/deleteWebhook
```

### База данных

База SQLite создаётся автоматически в папке `data/`. Миграции применяются при старте: добавление новых колонок происходит само.

### Пересборка нативных модулей

Если менялась версия Node.js, пересобери `better-sqlite3`:

```bash
npm rebuild better-sqlite3
```

## Безопасность

- `.env` не попадает в репозиторий — он в `.gitignore`.
- Токен бота и Telegram ID админа хранятся только в переменных окружения.
- Перед публикацией отзовите тестовый токен через @BotFather и создайте новый.

## Расширение проекта

Чтобы адаптировать бота под свой бизнес — создайте копию репозитория, поменяйте фото тортов, список изделий и переменные окружения. Все настройки собраны в `src/config/`.

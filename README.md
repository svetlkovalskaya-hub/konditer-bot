# Помощник кондитера — Telegram-бот для записи клиентов

Бот, который берёт на себя рутину записи клиентов на изготовление тортов, пряников, зефира и других десертов. Создан для домашних кондитеров и небольших кондитерских, у которых нет готовой CRM.

## Зачем этот бот

Домашний кондитер обычно ведёт заказы в блокноте, мессенджерах и голове. Это работает, пока заказов мало. Когда их становится больше, начинается хаос: забытые даты, двойные брони, переписка в трёх чатах, уточнения дизайна в одном месте, а адрес доставки — в другом.

Помощник кондитера собирает всё в одном окне Telegram:
- клиент выбирает изделие,
- выбирает удобную дату и время,
- указывает адрес доставки или самовывоз,
- добавляет комментарий и фото-примеры,
- получает подтверждение заказа.

А кондитер видит список заказов, статусы и детали одной командой.

## Что умеет бот

### Для клиента

- `/start` — приветствие и кнопка «Записаться».
- Выбор изделия из каталога.
- Inline-календарь для выбора даты.
- Готовые слоты времени или свободный ввод.
- Адрес доставки или самовывоз.
- Комментарий: дизайн, вкус, аллергены, пожелания.
- Прикрепление фото-примеров (до 5 штук).
- Предпросмотр заказа перед подтверждением.
- `/myorders` — список своих заказов.

### Для кондитера

- `/orders` — список последних заказов.
- `/order N` — детали заказа с фото.
- `/confirm N` — подтвердить заказ.
- `/cancel N` — отменить заказ.
- `/reschedule N YYYY-MM-DD` — перенести заказ на свободную дату.
- `/blockdate YYYY-MM-DD причина` — заблокировать дату.
- `/unblockdate YYYY-MM-DD` — разблокировать дату.
- `/blocked` — список заблокированных дат.
- `/addproduct` — добавить продукт в каталог.
- `/products` — список продуктов.
- `/setadmin ID` — назначить администратора.

## Как устроена защита от дублей

Все боты работают с одной базой данных SQLite. Перед сохранением заказа проверяется, свободна ли дата. Если дата занята другим заказом или заблокирована — бот предложит ближайшие свободные слоты. Так на одну дату не получится записаться дважды.

Архитектура сразу рассчитана на подключение второго бота — например, для Мессенджера MAX. Бизнес-логика вынесена в `orderService`, поэтому новый канал общения можно добавить, не переписывая ядро системы.

## Стек

- Node.js
- Express — мини-сервер для healthcheck и деплоя
- `node-telegram-bot-api` — Telegram-бот
- `better-sqlite3` — локальная база данных

## Локальный запуск

1. Скопируй `.env.example` в `.env`:
   ```
   cp .env.example .env
   ```

2. Заполни переменные:
   ```
   TELEGRAM_BOT_TOKEN=токен_из_@BotFather
   ADMIN_TELEGRAM_ID=твой_telegram_id
   PORT=3000
   NODE_ENV=development
   ```

3. Установи зависимости:
   ```
   npm install
   ```

4. Запусти:
   ```
   npm start
   ```

## Деплой на VPS (Beget и аналоги)

1. Купи VPS с Ubuntu 22.04 или 24.04.
2. Подключись по SSH:
   ```
   ssh root@IP_АДРЕС_СЕРВЕРА
   ```
3. Установи Node.js, npm, git:
   ```
   apt update && apt install -y nodejs npm git
   ```
4. Склонируй репозиторий:
   ```
   cd /opt
   git clone https://github.com/svetlkovalskaya-hub/konditer-bot.git
   cd konditer-bot
   npm install
   ```
5. Создай `.env`:
   ```
   nano .env
   ```
   Пример содержимого:
   ```
   NODE_ENV=production
   PORT=3000
   TELEGRAM_BOT_TOKEN=токен_из_@BotFather
   ADMIN_TELEGRAM_ID=твой_telegram_id
   ```
6. Установи PM2 и запусти бота:
   ```
   npm install -g pm2
   pm2 start src/index.js --name konditer-bot
   pm2 save
   pm2 startup
   ```
   Выполни команду, которую предложит `pm2 startup`.

## Управление ботом на сервере

```bash
cd /opt/konditer-bot

pm2 status          # статус бота
pm2 logs            # логи в реальном времени
pm2 logs --lines 50 # последние 50 строк логов
pm2 restart all     # перезапустить бота
pm2 stop all        # остановить бота
```

## Обновление кода

```bash
cd /opt/konditer-bot
git pull
npm install
pm2 restart all
```

## Деплой на Render

Render требует карту для диска и иногда нестабилен с VPN. Если всё же хочешь Render — инструкция в истории коммитов.

## Безопасность

- `.env` не попадает в репозиторий — он в `.gitignore`.
- Токен бота и Telegram ID админа хранятся только в переменных окружения.
- Перед публикацией рекомендуется отозвать тестовый токен через @BotFather и создать новый.

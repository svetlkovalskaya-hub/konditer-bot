# Скилл: работа с проектом «Помощник кондитера»

Этот скилл помогает Claude Code работать с репозиторием Telegram-бота для записи клиентов на кондитерские изделия.

## О проекте

Проект — это Node.js-приложение для автоматизации записи клиентов. Основной канал — Telegram-бот. Бот помогает клиентам выбрать изделие, дату, время, способ получения, оставить комментарий и фото. Кондитер управляет заказами через админ-панель в Telegram.

## Структура репозитория

```
├── src/
│   ├── index.js                  # точка входа: Express + запуск Telegram-бота
│   ├── config/
│   │   ├── index.js              # переменные окружения и настройки
│   │   └── cakeTypes.js          # список видов тортов с фото
│   ├── bot/telegram/
│   │   ├── index.js              # главный файл бота
│   │   ├── admin.js              # админ-панель
│   │   └── scenes/newOrder.js    # сценарий нового заказа
│   ├── db/index.js               # инициализация SQLite
│   ├── services/orderService.js  # бизнес-логика заказов
│   └── utils/
│       ├── date.js               # работа с датами
│       ├── keyboard.js           # клавиатуры Telegram
│       └── scheduler.js          # утреннее приветствие
├── data/                         # база SQLite
├── uploads/                      # фото от клиентов и фото тортов
│   └── cake-photos/              # фото для выбора вида торта
├── README.md                     # основная инструкция
├── .env.example                  # пример переменных окружения
└── package.json
```

## Как работать с кодом

### Перед изменениями

1. Проверь `.env` и убедись, что заданы `TELEGRAM_BOT_TOKEN` и `ADMIN_TELEGRAM_ID`.
2. Установи зависимости: `npm install`.
3. Проверь синтаксис изменённых файлов: `node --check путь/к/файлу.js`.
4. Для локальной проверки запускай: `npm start`.

### При внесении изменений

- Соблюдай существующий стиль: CommonJS, отступы в 2 пробела, короткие предложения в пользовательских сообщениях.
- Не удаляй файлы без согласования пользователя.
- Не выходи за пределы рабочей папки проекта.
- Папка `uploads/` и файл `.env` не должны попадать в git.
- Если меняешь структуру базы данных — добавь миграцию в `src/db/index.js` через проверку `PRAGMA table_info(orders)`.

### Обновление README

- README.md должен содержать актуальные команды и описание функций.
- Если добавляешь новую команду или админскую функцию — опиши её в README.
- Проверяй, что ссылки на Telegram используют формат `[текст](https://t.me/username)`.

### Проверка ошибок

1. После изменений запусти: `node --check` для всех JS-файлов в `src/`.
2. Проверь, что `npm install` отрабатывает без ошибок.
3. При проблемах с базой — проверь папку `data/` и пересобери `better-sqlite3` командой `npm rebuild better-sqlite3`.
4. Если бот не отвечает — проверь токен и убедись, что webhook сброшен.

### Подготовка к публикации

1. Проверь `README.md` на полноту и актуальность.
2. Убедись, что `.gitignore` скрывает `.env`, `data/`, `uploads/`, логи и базы данных.
3. Убедись, что `package.json` и `package-lock.json` актуальны.
4. Сделай коммит с понятным сообщением.
5. Запушь на GitHub: `git push origin main`.
6. При деплое на сервер не забудь скопировать фото тортов из `uploads/cake-photos/` — они не хранятся в git.

## Важные ограничения

- Бот работает через polling. Webhook не используется.
- Утреннее приветствие приходит в 9:00 по системному времени сервера.
- MAX-бот есть в коде, но запускается только при наличии `MAX_BOT_TOKEN`. В рамках этого скилла фокус на Telegram-боте.
- На одну дату можно принять ограниченное число заказов — настройка в `src/services/orderService.js`, константа `MAX_ORDERS_PER_DATE`.

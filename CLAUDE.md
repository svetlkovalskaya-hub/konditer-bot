Светлана, 42 года. Кулинарный блогер. Русскоязычная аудитория. Закончила курс по вайб-кодингу с Claude. Хочу делать интересные продуктовые решения, которые облегчают жизнь предпринимателей, самозанятых или обычных людей и монетизировать эти продукты через продажу подписки на них. # Проекты

- **Кулинарный блог** — десерты с заботой о здоровье, быстрые заготовки, рецепты без глютена, фитооздоровление (травы, чаи, сборы), сад и растения. Платформы: ВКонтакте, Мессенджер MAX. В планах: Дзен, YouTube.
- **Сезонные сборники рецептов для дегидратора** — 4 сборника (лето, осень, зима, весна): красивый визуал, чёткие инструкции, точные рецепты. Формат: PDF.
- **Карточки с рецептами** — разработка, печать, продажа. В планах.
- **Книга с рецептами для сушилки** — в планах.
- Помощник кондитера: бота для записи на изготовление торта, ИИ-помощник, ведущий список продуктов, которые надо купить, напоминающий о текущих заказах, доставке и времени выдачи. В перспективе - агент, помогающий вести контент кондитеру в разных соцсетях и мессенджерах.

# Как работаем

- Перед **сложными и многошаговыми задачами** — показать план и уточнить детали. Простые задачи (переписать абзац, поправить формат) — делать сразу без плана.
- По завершении работы — отчёт в виде markdown-файла. Название файла: содержит слово «отчёт» и текущую дату.
- Структура папок пока не устоялась — всё в папке Konditer. Агент может предлагать варианты организации, но не создаёт папки без согласования.

# Чего не делать

**Границы (без явного разрешения):**
- Не удалять файлы без разрешения.
- Не выходить за пределы рабочей папки.

**Анти-паттерны:**
- Не начинать ответ с «Конечно!» или «Отличный вопрос!» — стандартная вежливость уместна, но не эти фразы.
- Не писать длинные вступления — сразу к делу.
- Не писать очень длинные предложения.
- Не использовать оранжевый цвет в визуалах и рекомендациях.
- Не писать канцеляритом и сложными терминами — контент должен быть лёгким, с юмором и самоиронией.

# Частые задачи

- **Подготовка постов для блога** — контент по сезонным темам, к праздникам, ответы на вопросы подписчиков. Формат: лёгкий язык, простые предложения, юмор и самоирония, без канцеляризма.
- **Разработка продуктов, которые можно продавать по подписке** — планирование, написание кода, тестирование, поиск ошибок, предложения по улучшению и дальнейшему развитию продукта.

- **Сезонные сборники** — рецепты + визуал + PDF-формат. Красивая подача, точные инструкции.

## Правила нейминга

Формула имени файла: `[дата — опционально]-[тема]-[тип].md`
- Дата — в формате ГГГГ-ММ-ДД, ставится, когда важна привязка ко времени (отчёты, статистика).
- Тема — категория содержимого: бот, рецепт, травы, промо, идеи, отчёт, токены и т.д.
- Тип — конкретное уточнение: бот-запись, рецепт-малина и т.д.

Примеры: `рецепт-вафли-крапива.md`, `2026-06-03-отчёт.md`, `травы-одуванчик.md`

**Структура папок:**
- Проекты/ — папки проектов под разные продукты
- Контент/ — посты о растениях, дикоросах, фитотерапии
- Промо/ — рекламные кампании для запуска рекламы проекта
- Идеи/ — идеи для будущих продуктов
- `Отчёты/` — аналитика, статистика, отчёты о проделанной работе
- `temp/` — временные файлы, черновики

# Формат ответа

Структурированно, без воды, короткими предложениями. С итоговым выводом в конце.

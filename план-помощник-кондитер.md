# План: Помощник кондитера — MVP бот записи клиентов

**Дата плана:** 2026-06-27

## Что делаем

Telegram-бот, через который клиент записывается на изготовление десерта. Общая база данных и архитектура сразу готовятся под подключение MAX-бота.

## Архитектура

- **Backend:** Node.js + Express
- **База данных:** SQLite через `better-sqlite3` — не требует отдельного сервера, удобно для локальных тестов
- **Telegram-бот:** `node-telegram-bot-api` с polling (для локальной разработки проще всего)
- **MAX-бот:** отдельный модуль-заглушка с общим интерфейсом, подключаем к реальному API позже
- **Админка:** Telegram-команды для кондитера

## Функции MVP

### Для клиента

1. `/start` — приветствие и кнопка «Записаться»
2. Выбор продукта из каталога (торт, пряники, зефир и т.д.)
3. Выбор даты доставки/выдачи через inline-календарь
4. Выбор времени доставки/выдачи
5. Ввод адреса доставки или выбор самовывоза
6. Комментарий к заказу: дизайн, пожелания, аллергены
7. Прикрепление фото-примеров (до 5 штук)
8. Предпросмотр заказа и подтверждение
9. Уведомление админу о новом заказе

### Для кондитера (админа)

- `/addproduct` — добавить продукт
- `/products` — список продуктов
- `/orders` — список заказов
- `/order N` — детали заказа с фото
- `/confirm N` — подтвердить заказ
- `/cancel N` — отменить заказ
- `/blockdate YYYY-MM-DD` — заблокировать дату
- `/unblockdate YYYY-MM-DD` — разблокировать дату

## Схема базы данных

- `products` — каталог продуктов
- `orders` — заказы клиентов
- `order_photos` — фото, прикреплённые к заказам
- `blocked_dates` — даты, недоступные для записи
- `admins` — Telegram ID администраторов

## Как избежать дублей дат

- Все боты работают с одной базой данных
- Перед сохранением заказа проверяем, свободна ли дата
- Если дата занята другим заказом или заблокирована — предлагаем ближайшие свободные слоты
- Заказ фиксируется только после подтверждения клиентом

## Этапы реализации

### Этап 1. Скелет проекта
- Настроить `package.json`
- Создать структуру папок
- Добавить конфигурацию и `.env.example`

### Этап 2. База данных
- Создать схему SQLite
- Добавить seed-данные с примерами продуктов
- Написать методы для работы с заказами и проверки занятости дат

### Этап 3. Telegram-бот для клиента
- Подключить polling
- Реализовать сценарий записи
- Сохранять фото в папку и записывать ссылки в базу

### Этап 4. Команды администратора
- Управление продуктами
- Просмотр и подтверждение заказов
- Блокировка и разблокировка дат

### Этап 5. Подготовка под MAX
- Вынести бизнес-логику в общие модули
- Сделать интерфейс для MAX-бота
- Написать инструкцию, что менять при появлении токена MAX

### Этап 6. Тестирование
- Запустить локально
- Проверить полный сценарий заказа
- Проверить защиту от двойной записи на одну дату

## Ограничения MVP

- Без онлайн-оплаты
- Без автоматических напоминаний клиентам
- Без веб-панели администратора
- MAX-бот не подключён к реальному API, но архитектура готова

## Что нужно от тебя

1. Создать тестового Telegram-бота через @BotFather и прислать токен
2. Прислать свой Telegram ID для назначения администратором
3. Дать список продуктов для тестового каталога

## Результат

Работающий локально Telegram-бот для записи клиентов. Код с общей логикой, готовой к подключению MAX. Отчёт о проделанной работе.

# Отчёт: бот "Помощник кондитера" работает на VPS

**Дата:** 2026-07-02

## Результат

- Бот задеплоен на VPS Beget (Ubuntu 24.04).
- Бот работает круглосуточно через PM2.
- Клиенты могут записываться через Telegram.
- Заказы сохраняются в базе данных SQLite.
- Кондитер получает уведомления о новых заказах в Telegram.
- Админ-панель полностью переведена с команд на кнопки.

## Что сделано

### 1. Репозиторий на GitHub

- https://github.com/svetlkovalskaya-hub/konditer-bot
- Код бота, README, конфигурация, отчёты, документы проекта.
- `.env` с токеном не попал в репозиторий.
- На GitHub актуальная версия кода.

### 2. VPS на Beget

- Куплен сервер с Ubuntu 24.04.
- Установлены Node.js, npm, git.
- Создан SSH-ключ для удалённого управления.

### 3. Запуск бота

- Репозиторий склонирован в `/opt/konditer-bot`.
- Установлены зависимости `npm install`.
- Создан `.env` с переменными окружения.
- Бот запущен через PM2.
- Настроен автозапуск PM2 при перезагрузке сервера.

### 4. Админ-панель

Кнопки, которые видит кондитер:

- 📋 Список заказов
- 🔍 Найти по имени
- 📞 Поиск по телефону
- ❌ Удалить заказ
- 🔄 Перенести запись
- 🏠 Общее меню

Особенности:

- Список заказов выводится по дате, ближайшие сверху.
- Каждый заказ — отдельное сообщение с эмодзи 🎂.
- Формат: дата и время, изделие, имя клиента, телефон, номер заказа, статус.
- Удалённые заказы не показываются в списке.
- Поиск работает по имени и по телефону.
- Удаление — мягкое, заказ переводится в статус `cancelled`.
- Перенос записи работает по номеру заказа и новой дате.

### 5. Исправленные ошибки

- **Ошибка подключения к Telegram**: локальный VPN HAPP блокировал polling. Решение — перенос на VPS.
- **Ошибка `resetSession is not a function`**: функция не экспортировалась. Исправлен экспорт.
- **Ошибка `sendMainMenu`**: передавался объект бота вместо chatId. Исправлен вызов.
- **Ошибка `SQLITE_CONSTRAINT_NOTNULL`**: в сессии не сохранялся `clientId`. Исправлено.
- **Статус `pending` вместо `создан`**: в базе данных и в коде статус по умолчанию теперь `создан`.
- **Список заказов показывал удалённые**: теперь `getOrders` исключает статус `cancelled`.

### 6. Очистка

- Удалены 6 тестовых заказов с сервера (переведены в `cancelled`).
- Очищены логи PM2.
- Удалены временные файлы: скриншоты, старый конфиг `fly.toml`, дубль аватарки.
- В репозиторий добавлены документы проекта: отчёты, план, описание, идеи.

### 7. Проверка

- `/start` работает.
- Полный сценарий заказа работает.
- Заказ создаётся после нажатия «Да» на предпросмотре.
- Уведомление админу приходит в Telegram.
- Админ-панель протестирована пользователем — всё работает хорошо.

## Как управлять ботом

Подключись к серверу:

```bash
ssh root@159.194.204.27
```

Команды:

```bash
cd /opt/konditer-bot
pm2 status          # статус
pm2 logs            # логи
pm2 logs --lines 50 # последние 50 строк
pm2 restart all     # перезапуск
pm2 stop all        # остановка
```

## Как обновить код

```bash
cd /opt/konditer-bot
git pull
npm install
pm2 restart all
```

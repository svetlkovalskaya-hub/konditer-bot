# Отчёт: Помощник кондитера — MVP бот записи клиентов

**Дата:** 2026-06-27

## Что сделано

Создан MVP Telegram-бота для записи клиентов на изготовление десертов. База данных и бизнес-логика сразу ориентированы на подключение второго бота (MAX) в будущем.

## Результат

### Работающий код

- Telegram-бот с полным сценарием записи.
- SQLite-база данных.
- Админские команды для управления заказами.
- Защита от двойной записи на одну дату.
- Заглушка для MAX-бота с общей бизнес-логикой.
- Подготовка к деплою на Render.

## Сценарий клиента

1. `/start` → кнопка «Записаться».
2. Выбор изделия: Торт, Пряники, Зефир, Макарон, Капкейки.
3. Выбор даты через inline-календарь.
4. Выбор времени (готовые слоты или своё время).
5. Доставка или самовывоз.
6. Комментарий к заказу.
7. Прикрепление фото-примеров (до 5 штук).
8. Предпросмотр и подтверждение.
9. Уведомление администратору о новом заказе.
10. Просмотр своих заказов через `/myorders`.

## Команды администратора

| Команда | Описание |
|---------|----------|
| `/orders` | Список последних заказов |
| `/order N` | Детали заказа с фото |
| `/confirm N` | Подтвердить заказ |
| `/cancel N` | Отменить заказ |
| `/reschedule N YYYY-MM-DD` | Перенести дату заказа |
| `/blockdate YYYY-MM-DD причина` | Заблокировать дату |
| `/unblockdate YYYY-MM-DD` | Разблокировать дату |
| `/blocked` | Список заблокированных дат |
| `/addproduct` | Добавить продукт |
| `/products` | Список продуктов |
| `/setadmin ID` | Назначить администратора |

## Защита от дублей дат

- Все боты (сейчас Telegram, позже MAX) работают с одной базой.
- Перед сохранением заказа проверяется свободность даты.
- Если дата занята или заблокирована — клиент получает ближайшие свободные даты.
- Админ может перенести заказ на другую дату, если она свободна.

## Тестирование

### Что проверено

- Бот запускается без ошибок.
- База данных создаётся автоматически.
- Продукты добавляются из seed-данных.
- Создание двух заказов на одну дату блокируется.
- Генерируются альтернативные свободные даты.

### Что не удалось протестировать

- Реальный обмен сообщениями с Telegram.
- Причина: на текущем компьютере нет доступа к `api.telegram.org` даже с включённым VPN. Curl и Node.js не могут установить соединение.

### Лог ошибки


## Следующий шаг

Задеплоить бота на Render. Там связь с Telegram будет работать без VPN, и бот будет онлайн круглосуточно.

Инструкция по деплою:

1. Создать аккаунт на https://render.com.
2. Загрузить проект на GitHub.
3. В Render создать New Web Service из репозитория.
4. Build Command: `npm install`
5. Start Command: `npm start`
6. Добавить переменные окружения:
   - `NODE_ENV=production`
   - `TELEGRAM_BOT_TOKEN=YOUR_BOT_TOKEN_HERE`
   - `ADMIN_TELEGRAM_ID=YOUR_TELEGRAM_ID_HERE`
7. Добавить диск:
   - Name: `konditer-data`
   - Mount Path: `/data`
   - Size: 1 GB
8. Нажать Deploy.

Файл `render.yaml` уже подготовлен для автоматической настройки.

## Важные замечания

- Токен Telegram-бота и Telegram ID админа хранятся в `.env`. Файл `.env` добавлен в `.gitignore`.
- Для продакшена рекомендуется отозвать текущий токен через @BotFather и создать новый.
- Папка `data/` содержит тестовую базу SQLite. Можно оставить для разработки или удалить перед деплоем.
